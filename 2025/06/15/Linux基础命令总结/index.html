Linux基础命令总结

Linux目录结构

  • Linux没有盘符的概念，只有一个根目录
  • 路径之间的层级关系用/来表示，例如/usr/local/hello.txt

Linux命令基础

通用的格式：

  • command 命令本身
  • -options 命令的一些选项，控制细节
  • parameter 命令的参数，多数用于命令的指向目标等

后面两个都是可选。

ls

作用：列出当前工作目录下的文件和子目录。默认当前工作目录是HOME目录。HOME目录是每一个用户在Linux的专属目录。

  • -a列出全部的文件，包括隐藏文件和文件夹
  • -l以列表的形式纵向列出所有的内容，包含日期等更多的信息
  • -h用更易于阅读的形式列出文件的大小，与-l一起使用，如果这时还没有单位那就是默认的字节
  • 支持混用。例如-al -l-a

注：ls -l可以简写为ll

cd pwd

cd [path]: Change Directory

不给参数就回到HOME目录。

pwd: Print Work Directory，作用：查看当前的工作目录的位置

find 起始路径 -name 被查找的文件名

用户和权限

拥有最大权限的用户的账户名是root，其他的属于普通用户。

如何切换用户

切换到root用户后，可以使用exit回退到上一个用户，或者快捷键ctrl+d。

临时获得root权限：sudo。并不是所有的用户都可以使用sudo，必须为普通用户添加sudo认证。

Linux关于权限的管控级别：

  • 针对用户的权限控制
  • 针对用户组的权限控制

针对特定的文件/目录，既可以控制用户权限，也可以控制用户组的权限。

软硬链接

什么是软链接？

在系统中的软连接可以把文件、文件夹链接到其他位置，类似Windows操作系统中的快捷方式，也叫符号链接（Symbolic Link）。

语法：

  • -s：创建软连接 soft
  • 参数一：被链接的文件/文件夹
  • 参数二：链接的目的地

默认情况下，ln命令产生硬链接，要搞懂硬链接，首先要明白什么是索引节点号：

硬连接指通过索引节点来进行连接。在 Linux 的文件系统中，保存在磁盘分区中的文件不管是什么类型都给它分配一个编号，称为索引节点号(Inode Index)。在 Linux 中，多个文件名指向同一索引节点是存在的。比如：A 是 B 的硬链接（A 和 B 都是文件名），则 A 的目录项中的 inode 节点号与 B 的目录项中的 inode 节点号相同，即一个 inode 节点对应两个不同的文件名，两个文件名指向同一个文件，A 和 B 对文件系统来说是完全平等的。删除其中任何一个都不会影响另外一个的访问。

硬链接相当于原文件的替身，不过它拥有和原文件相同的索引节点号，代表的是同一个东西。一个文件对应一个唯一的索引节点号，但是可以拥有多个替身，只要还有替身存活，这个文件就还存在的（对于静态文件（没有进程正在调用），当硬链接数为0时文件就被删除）。换句话说，只要这个文件还在被引用，那么它就是存活的。（类似于JVM的强引用机制）

软连接是一个新的文件，它保存了源文件的路径，如果源文件被删除了，软链接还在，但是已经没用了。

目录可以拥有软链接，但是不能有硬链接，这是为了防止出现目录循环，例如：

这时如果对其中一个目录调用ls -R，会陷入无限循环。

另一方面，Linux的目录是树结构，如果允许硬链接的话会变成图甚至有环图，文件系统会很难维护，这是不能接受的。因此，ln命令禁止对目录创建硬链接。

Linux的端口

共65535个端口，分为三类；

  • 公认端口：1~1023，不能随便使用
  • 注册端口：1024~49151，可以用来绑定一些程序
  • 动态端口：49152~65535，通常不会固定绑定程序，当程序对外进行网络连接时，用于临时使用

僵尸进程：

一个子进程已经完成执行并退出，但是它的父进程没有调用 wait() 或 waitpid() 来回收它的资源，导致它在系统的进程表中残留一个条目，状态为“僵尸”。

危害：僵尸进程数量过多可能会耗尽PID表，系统无法再创建新进程。

解压缩

  • .tar称之为tarball，归档文件，只是把文件组装到一个.tar文件内，并没有太多文件体积的减少。
  • .gz，也有.tar.gz，以gzip格式压缩算法，可以极大减少压缩后的体积。

tar命令可以进行解压缩的操作

  • -c创建压缩文件。用于压缩
  • -v显示压缩和解压的进度
  • -x解压模式
  • -f放在最后，指名后面是要解压缩的文件
  • -z指名操作算法为gzip，不写的话就是tarball
  • -C选择解压的目的地，用于解压模式，单独使用，和其他参数分开

压缩常用：tar -cvf test.tar 文件，或者tar -zcvf test.tar.gz 文件

解压常用：tar -xvf test.tar -C 目录名，或者tar -zxvf test.tar.gz -C 目录名

SFTP

SFTP，全称是 SSH File Transfer Protocol（安全文件传输协议），是 Linux 系统中最常用的一种 通过 SSH 安全加密进行文件传输的协议。它既安全又稳定，用于在本地计算机与远程服务器之间上传和下载文件。

SFTP可以用户windows系统和Vmware中的Linux系统互传文件。MobaXterm内嵌Sftp模块，可以很方便地进行上传或者下载文件（支持拖拽）。注意，SFTP文件管理器是通过初始登录时用的用户名来访问远程文件的，如果登陆的时候是以普通用户的身份登录，那么SFTP无法处理root权限目录下的内容，除非将该目录权限设置为777。
